# data/ — 数据操作规范

## 目录结构

```
data/
├── raw/          # 原始数据（只读，赛题附件直接放这里）
├── processed/    # 清洗后数据（预处理脚本输出到此处）
└── external/     # 参考数据（往年数据、外部公开数据）
```

## 规范

- **原始数据不改写**：`raw/` 里的文件是赛题附件的原始副本，脚本只读不写
- **清洗脚本记录变换**：任何清洗操作（缺失值填充、异常值处理、格式转换）都应该记入日志，说明为什么这样做
- **处理后的数据用命名标识处理步骤**：如 `wind_cleaned.csv`, `load_normalized.xlsx`
- **非必要不生成中间文件**：能用内存处理的尽量不写到磁盘，减少混乱
